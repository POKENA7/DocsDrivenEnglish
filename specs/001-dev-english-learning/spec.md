# Feature Specification: プログラマー向け英語学習サイト（MVP）

**Feature Branch**: `001-dev-english-learning`  
**Created**: 2026-01-03  
**Status**: Draft  
**Input**: User description: "プログラマー向け英語学習サイト：技術ドキュメントを教材に単語/読解クイズと学習履歴を提供（MVP）"

## Clarifications

### Session 2026-01-03

- Q: 教材化するコンテンツ取得範囲は？ → A: 入力URLの「その1ページのみ」を教材化（crawlしない）
- Q: ログイン方式は？ → A: Clerk でログイン
- Q: 出題の出典表示は？ → A: 引用テキスト + 元URL（入力URLそのもの）を表示
- Q: 1セッションの問題数は？ → A: 10問（完了後に続行可能）
- Q: 問題が10問作れない場合は？ → A: ある分だけ出題し、10問未満なら「全X問で終了」扱いにする

## User Scenarios & Testing *(mandatory)*

### User Story 1 - ドキュメントURLからクイズ学習する (Priority: P1)

ユーザーはトップページで公式ドキュメントのURLを入力し、学習モード（単語モード / 読解モード）を選択して、4択クイズに回答できる。回答後に正誤と解説（英語としての意味＋技術的な背景＋使用シーン）を確認し、次の問題へ進める。

**Why this priority**: 本サービスの価値（英語学習と技術学習の同時体験）を最短で提供できるため。

**Independent Test**: 公開されているドキュメントURLを1つ入力し、10問連続で「出題→回答→解説→次へ」を完走できることを確認できる。

※ただし、抽出できる問題が10問未満の場合は「全X問で終了」までを完走できること。

**Acceptance Scenarios**:

1. **Given** ユーザーがトップページにいる, **When** 有効なURLを入力して学習開始を選ぶ, **Then** モード選択画面が表示される
2. **Given** ユーザーが学習モードを選択済み, **When** 問題に回答して確定する, **Then** 正誤と解説が表示され、次の問題へ進める

---

### User Story 2 - 未ログインでも学習を続けられる (Priority: P2)

ユーザーが未ログインでも学習は可能で、セッション内では学習状況（学習した問題数、正答率）が更新される。学習履歴画面では、未ログインの場合は「ログインすると履歴を保存できる」ことが分かる。

**Why this priority**: 学習開始のハードルを下げ、初回体験を損なわないため。

**Independent Test**: 未ログイン状態で複数問回答し、セッション中は学習状況が表示され、学習履歴画面でログイン誘導が表示されることを確認できる。

**Acceptance Scenarios**:

1. **Given** ユーザーが未ログイン, **When** 5問回答する, **Then** セッション内の「学習した問題数」と「正答率」が反映される
2. **Given** ユーザーが未ログイン, **When** 学習履歴画面を開く, **Then** 永続化されない旨とログイン誘導が表示される

---

### User Story 3 - ログインで学習履歴が永続化される (Priority: P3)

ユーザーがログインすると、学習履歴がアカウントに紐づいて保存され、複数デバイスから参照できる。

**Why this priority**: 継続学習の動機づけ（進捗可視化、複数デバイス利用）を実現するため。

**Independent Test**: ログイン状態で学習→履歴表示までを行い、再ログイン後も履歴が残っていることを確認できる。

**Acceptance Scenarios**:

1. **Given** ユーザーがログイン済み, **When** 学習して10問回答する, **Then** 学習履歴画面に結果が保存・表示される
2. **Given** ユーザーが一度ログアウトした, **When** 再度ログインして学習履歴画面を開く, **Then** 過去の学習結果が参照できる

---

### Edge Cases

- URLが不正（空文字、http/https以外、明らかにURLではない文字列）の場合はどうなるか？
- 入力URLのページが取得できない（ページが存在しない、タイムアウト、アクセス制限）場合はどうなるか？
- 取得できても教材化できる本文がほとんどない（テーブル/コードのみ等）場合はどうなるか？
- 抽出結果が極端に少ない場合に、出題をどう継続/停止するか？
- 同じ単語・同じ文が繰り返し出題されないようにできるか？
- 未ログインでブラウザ/タブを閉じた場合に、セッション履歴が消えることがユーザーに誤解なく伝わるか？

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: System MUST ユーザーが公式ドキュメントのURLを入力し、学習開始できる
- **FR-002**: System MUST URLの形式を検証し、不正な入力に対して学習開始を防止し、理由が分かるエラー表示を行う
- **FR-003**: System MUST 入力されたURLのドキュメント本文を教材として取り込み、出題に利用できる学習素材（単語/表現、英文）を生成する
- **FR-004**: System MUST 教材本文に出現する英単語・英語表現のうち、技術ドキュメントで頻出する語やプログラミング文脈で意味が重要になる語を優先して出題する
- **FR-005**: System MUST 技術的意味を持たない機能語（助動詞・前置詞・冠詞など）は出題対象から除外する（例: can, will, the, a, to, of）
- **FR-006**: System MUST 学習開始時に、単語モード/読解モードをユーザーが選択できる
- **FR-007**: System MUST 単語モードで、英単語/表現の意味を問う4択問題を出題する（選択肢は常に4つ）
- **FR-008**: System MUST 読解モードで、ドキュメント由来の英文（1文または短い段落）の意味理解を問う4択問題を出題する（選択肢は常に4つ）
- **FR-009**: System MUST 回答確定後に、正誤表示と解説を表示する
- **FR-010**: System MUST 解説に次の要素を含める: 英語としての意味、技術的な背景/注意点、実際の使用シーン
- **FR-011**: System MUST 出題の根拠として、引用テキストと元URL（入力URL）をユーザーが確認できるようにする
- **FR-012**: System MUST 未ログインユーザーの学習を許可し、学習履歴はセッション内のみ保持する
- **FR-013**: System MUST Clerk によるログイン/ログアウトを提供し、ログインユーザーの学習履歴を永続化する
- **FR-014**: System MUST ログインユーザーの学習履歴を複数デバイスから参照できる
- **FR-015**: System MUST 学習履歴として「学習した問題数」「正答率」「継続学習日数」をユーザーに表示する（継続学習日数はログインユーザーを対象とする）
- **FR-016**: System MUST 学習履歴画面を提供し、未ログイン時はログイン誘導を表示する
- **FR-017**: System MUST MVPのスコープとして、教材化の対象はユーザーが入力したURLの「その1ページ内容」に限定する（リンクを辿るcrawlは行わない）
- **FR-018**: System MUST 学習セッションを5問単位で完了でき、完了後にユーザーが続行して次の5問を開始できる
- **FR-019**: System MUST 抽出できる問題数が5問未満の場合、ある分だけ出題し「全X問で終了」としてセッションを完了できる

### Assumptions

- ユーザーは、公開されていて取得可能な公式ドキュメントURLを入力する
- 画面表示・選択肢・解説は日本語、教材（問題文）は英語を基本とする
- 学習対象を切り替えたい場合、ユーザーは別のURLを入力して学習を開始する
- 「継続学習日数」は、同一ユーザーが別日付に学習した日数として集計される

### Dependencies

- 学習時点で入力URLへアクセスでき、教材として利用可能な本文が取得できること
- 一般的なインターネット接続環境があること
- Clerk の認証が利用可能であること

### Acceptance Criteria (Requirements-level)

1. **Given** ユーザーが不正なURLを入力した, **When** 学習開始を選ぶ, **Then** 学習は開始されず、理由が分かるエラーが表示される
2. **Given** ユーザーが有効なURLを入力した, **When** 学習開始を選ぶ, **Then** 単語モード/読解モードの選択ができる
3. **Given** ユーザーが単語モードで学習中, **When** 問題が表示される, **Then** 選択肢は常に4つである
4. **Given** ユーザーが読解モードで学習中, **When** 問題が表示される, **Then** 選択肢は常に4つである
5. **Given** ユーザーが回答を確定した, **When** 結果が表示される, **Then** 正誤と、英語としての意味/技術的背景/使用シーンを含む解説、および（引用テキスト + 元URL）が表示される
6. **Given** ユーザーが未ログイン, **When** 学習して履歴画面を開く, **Then** セッション内の集計のみが表示され、永続化のためのログイン誘導が表示される
7. **Given** ユーザーが未ログイン, **When** Clerk でログインする, **Then** ログイン状態になり、履歴の永続化が有効になる
8. **Given** ユーザーがログイン済み, **When** 学習してから再ログインする, **Then** 過去の学習履歴が保持され参照できる
9. **Given** 教材本文に機能語（助動詞・前置詞・冠詞など）が含まれる, **When** 出題が行われる, **Then** それらは出題対象として選ばれない
10. **Given** ユーザーが学習セッション中, **When** 5問目の解説を確認する, **Then** セッション完了が分かり、続行して次の5問を開始できる
11. **Given** ユーザーが学習セッション中, **When** 抽出できる問題が5問未満で最終問題の解説を確認する, **Then** 「全X問で終了」が分かり、学習が完了できる

### Key Entities *(include if feature involves data)*

- **User**: ログイン状態を持つユーザー（匿名ユーザーを含む）
- **Study Session**: 5問単位の学習セッション（inputUrl/mode、出典用sourceUrl/sourceQuoteText、開始/完了時刻など）
- **Question**: 出題単位（sessionに紐づくモード、問題文、選択肢、正解、出典）
- **Attempt**: 1問に対する回答結果（正誤、回答時刻、対象Question）
- **Study Summary**: 学習履歴の集計（問題数、正答率、継続学習日数）

Notes:
- MVPではユーザー入力URLをその場で教材として扱うため、"Learning Corpus" の永続化は行わない（必要に応じて将来 cache 用の "Document Source" を追加可能）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 有効なURL入力後、ユーザーが学習開始を選んでから最初の問題が表示されるまで3秒以内である
- **SC-002**: 初回ユーザーが、単語モードまたは読解モードで（最大）5問の学習を中断せず完了できる（タスク完了率80%以上）
- **SC-003**: 学習セッション内で、出題→回答→解説閲覧の一連フローが5回連続で成立する（エラーにより継続不能にならない）
- **SC-004**: ログインユーザーが別デバイスでログインした際に、過去の学習履歴を参照できる（再現率100%）
