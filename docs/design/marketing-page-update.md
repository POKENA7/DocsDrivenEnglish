# マーケティングトップページ更新設計書

## 背景・目的

現在の `/` ページ（`(marketing)/page.tsx`）は旧仕様（"URL を貼るだけで公開ドキュメントから問題を生成"）をそのまま表示しており、現在の仕様と乖離している。仕様書（`docs/SPEC.md`）に合わせてコピー・ステップ説明を更新する。

---

## 現状 vs 仕様の差分

| 箇所 | 現在の表示 | 正しい仕様 |
|------|-----------|-----------|
| キャッチコピー（バッジ） | `公開ドキュメント → 教材 → クイズ` | `トピック → クイズ → 復習` |
| サービス説明文 | "URL を貼るだけで、1ページ分の公開ドキュメントから問題を生成し、最大5問のクイズで学習します。" | "学習したい技術トピックを入力するだけで AI が英語クイズを生成。word / reading の2モード、最大20問まで出題できます。" |
| Step 01 タイトル | URLを入力 | トピックを入力 |
| Step 01 説明 | 公開ドキュメント（1ページ）を指定。 | 学習したい技術要素（例: React Hooks、Kubernetes Pod）をテキストで入力。 |
| Step 02 タイトル | 問題を生成 | モードと問題数を設定 |
| Step 02 説明 | word / reading を選べます。 | word / reading モードと出題数（1〜20問）を選択。他のユーザーのクイズから出題も可能。 |
| Step 03 タイトル | 解いて定着 | 解いて・復習して定着 |
| Step 03 説明 | 解説と Source を確認できます。 | 解説・「もっと解説」で理解を深め、不正解問題は自動で復習キューへ登録されます。 |

---

## 変更対象ファイル

- `frontend/src/app/(marketing)/page.tsx`

---

## 変更方針

- UI 構造（カード3枚のグリッド、ボタン配置）はそのまま維持する
- テキストコピーのみ更新する
- ログイン必須であることはトップページでは強調しない（UX的に冗長なため）
- 「他のユーザーのクイズ」は Step 02 の補足として1文で触れる程度に留める
- 復習機能は Step 03 で簡潔に説明する

---

## 変更後コピー詳細

### バッジ
```
トピック → クイズ → 復習
```

### サービス説明文
```
「React Hooks」「Kubernetes Pod」など、気になる技術トピックを入力するだけ。AI が本物の技術ドキュメントさながらの英語クイズをその場で生成します。毎日続けて、技術英語をサクッとモノにしよう。
```

### Step 01
- タイトル: `トピックを入力`
- 説明: `学びたい技術要素をテキストで入力するだけ。難しい準備は一切なし。`

### Step 02
- タイトル: `モードと問題数を設定`
- 説明: `word / reading の2モード＆最大20問から自由にカスタマイズ。他のユーザーのクイズにも挑戦できます。`

### Step 03
- タイトル: `解いて・復習して定着`
- 説明: `丁寧な解説＋「もっと解説」で深掘り。不正解問題は自動で復習キューに登録されるから、忘れた頃にもう一度。`

---

## テスト・確認方針

- UI の目視確認（`pnpm run preview`）
- lint / format:check を通すこと（テキスト変更のみのため、ロジックテストは不要）
