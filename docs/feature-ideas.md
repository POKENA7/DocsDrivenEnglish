# 機能アイデア一覧

**作成日**: 2026-02-18

このドキュメントは、DocsDrivenEnglish に追加すると学習効果・継続率が向上すると考えられる機能のアイデアをまとめたものです。  
優先度（★の数）は「あった方が良い度」を表し、★★★が最優先です。

---

## 機能一覧

### ★★★ 1. 単語帳・間違い復習機能

間違えた問題や「気になった」とマークした問題を後から集中して復習できる機能。

**なぜ重要か**  
英語学習の根幹は「繰り返し」にある。1回のクイズで間違えた単語を流してしまうと記憶に定着しない。間違い問題を専用の「単語帳」としてストックし、スペースド・リペティション（間隔反復）で再出題することで、記憶定着率を大幅に改善できる。

**主な要件**
- 回答後に「単語帳に追加」ボタンで問題をブックマーク
- 単語帳一覧ページで保存済み問題を確認
- 「単語帳から復習」モードで保存問題のみでクイズ実施
- 正解できた問題を単語帳から自動的に卒業させる（オプション）

**設計書**: [design/flashcard-review.md](design/flashcard-review.md)

### ★★★ 2. 学習ストリーク・ゲーミフィケーション

毎日学習を続けると「ストリーク（連続学習日数）」が伸び、XP（経験値）を獲得できる仕組み。

**なぜ重要か**  
英語学習の最大の障壁は「継続」である。ゲーム的な要素を加えることで「今日もやらないとストリークが切れる」という動機付けが生まれ、習慣化を促進できる。Duolingo をはじめとした学習アプリが実証済みの手法。

**主な要件**
- 連続学習日数（ストリーク）の計算・表示
- 1問正解ごとにXP獲得、累計XPに応じたレベル表示
- ストリークが途切れそうな場合のリマインダー通知（メール/ブラウザ通知）
- 週間・月間の学習カレンダー表示

**設計書**: [design/streak-gamification.md](design/streak-gamification.md)

### ★★★ 3. 弱点分析ダッシュボード

間違いの傾向をカテゴリ・難易度・技術トピック別に可視化する分析画面。

**なぜ重要か**  
闇雲に学習するより「自分がどこでつまずいているか」を把握することで学習効率が劇的に向上する。「React の用語系が弱い」「読解モードの正答率が低い」といったインサイトをユーザーに提供することで、次に学ぶべきトピックの選択にも活用できる。

**主な要件**
- モード別（単語/読解）正答率グラフ
- 技術トピック別の正答率ランキング（得意 / 苦手トピック可視化）
- 間違いの多い問題一覧（上位10件）
- 時系列での正答率推移グラフ

**設計書**: [design/weakness-analysis-dashboard.md](design/weakness-analysis-dashboard.md)

### ★★ 4. 音声読み上げ（TTS）機能

英単語・英文をブラウザの Text-to-Speech API またはサーバー側 TTS API で音声再生する機能。

**なぜ重要か**  
技術英語は読めても発音を知らないケースが多い（例: "albeit", "daemon", "cache"）。音声で正しい発音を確認することで、実際の英語コミュニケーションや会議にも役立つ知識になる。

**主な要件**
- 問題文・解説の英語テキスト横に「再生」ボタンを配置
- Web Speech API（`SpeechSynthesis`）を使ったクライアントサイド再生（追加コストなし）
- 再生速度の調整（0.75x / 1.0x / 1.25x）
- 設定でON/OFFを切り替え可能

### ★★ 5. お気に入りトピック・プリセット機能

よく使う技術トピックを「お気に入り」として保存したり、運営が用意したプリセットトピックから選んで学習を開始できる機能。

**なぜ重要か**  
毎回「React Hooks」と入力するのは手間。お気に入り保存によりリピート学習のハードルが下がる。初心者向けにはプリセットトピック（「JavaScript 基礎」「Docker」「Git」など）を用意することで、何を入力すれば良いか迷う問題も解消できる。

**主な要件**
- 学習後に「このトピックをお気に入り登録」ボタンを表示
- `/learn` 画面にお気に入り一覧をワンタップで選択できる形で表示
- 運営がキュレーションした「おすすめトピック」セクション（フロントエンド / バックエンド / インフラ / AI カテゴリ）

---

### ★★ 6. 難易度アダプティブ出題

ユーザーの直近の正答率に応じて、LLM に生成させる問題の難易度をリアルタイムで調整する機能。

**なぜ重要か**  
正答率が高すぎると退屈し、低すぎると挫折する。ユーザーの習熟度に合わせた問題を出すことで、学習の「ゾーン」を維持できる（フロー理論）。

**主な要件**
- 直近N問の正答率が 80% 超 → LLM プロンプトに「より高難度の問題を生成」指示を付加
- 直近N問の正答率が 50% 未満 → LLM プロンプトに「基本的な問題を生成」指示を付加
- 難易度の目安をユーザーに表示（「初級 / 中級 / 上級」バッジ）

### ★★ 7. スペル入力クイズモード

4択から選ぶ現行形式に加え、英単語を直接キーボード入力する記述式モードを追加する。

**なぜ重要か**  
4択は「なんとなく見覚えがある」だけで正解できるため、再現性のある記憶定着がしにくい。スペル入力を求められることで、アクティブリコール（能動的記憶想起）が促進され、定着率が上がる。

**主な要件**
- 単語モードでのみ選択可能な「スペル入力モード」オプション
- タイプ時のリアルタイムフィードバック（文字数ヒント、部分一致表示）
- 大文字/小文字は区別しない、ハイフンは許容する等の採点ルール設定
- 4択モードより高いXP倍率を設定（難しいモードへのインセンティブ）

---

### ★ 8. 学習リマインダー通知

設定した時刻や間隔でブラウザプッシュ通知またはメール通知を送る機能。

**なぜ重要か**  
ユーザーが能動的にアプリを開く習慣がつくまでのサポートとして機能する。毎朝9時に「今日の英単語クイズに挑戦！」と通知するだけで継続率が改善する。

**主な要件**
- ブラウザ Push Notifications（Web Push API）
- 通知時刻・頻度をユーザーが設定
- ストリークが途切れそうな場合（23時台）の緊急リマインダー

---

### ★ 9. SNS シェア・ランキング機能

学習結果をSNSにシェアしたり、同じトピックを学んだユーザー間でスコアを競う機能。

**なぜ重要か**  
「今日○問正解した！」を SNS でシェアすることが学習の報告・継続動機になる。また、友人間でのランキングは競争心を刺激し、学習のモチベーション維持に効果的。

**主な要件**
- セッション完了画面に「X（Twitter）でシェア」ボタン
- OGP 画像生成（正答率・トピック名を含む画像）
- オプショナルな週間ランキング（ログインユーザー対象）

---

### ★ 10. 英文タイピング練習モード

問題の英文をそのまままったく同じにタイピングする練習。正確さと速度（WPM）を計測する。

**なぜ重要か**  
タイピングしながら英文を読む行為そのものが英語習得に効果的（写経効果）。さらに技術英語の「正確なスペル」を意識する習慣がつき、コーディング中のコメント・ドキュメント記述にも活用できる。

**主な要件**
- クイズセッション後に「この英文をタイプして練習」オプションを表示
- タイピング速度（WPM）と正確さ（精度%）を計測・表示
- ミスタイプした文字のリアルタイムハイライト
